# issues（草稿）

尚未在 GitHub 上发帖时，把 **Issue / Discussion 正文草稿** 放在本目录，便于 PR 里链回单一事实来源；发帖后在 Issue 里贴 **Trigger comment** URL。

在 Cursor 中已启用 **GitHub MCP** 时，Agent 应优先用 MCP 创建/同步线上 Issue 与 Discussion，再把 URL 回填 `doc/tasks/`（见 [[GITHUB_MCP]]）。
